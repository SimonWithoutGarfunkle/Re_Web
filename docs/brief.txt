Brief de Développement : Landing Page & Auth UI "RE:"
Objectif : Développer la vitrine technologique du projet "RE:" en React, en utilisant une bibliothèque de composants professionnelle pour garantir l'unité de la charte graphique (DA) et la réutilisabilité du code.
1. Stack Technique Imposée
Framework : React (Vite.js).
UI Library : MUI (Material UI) v6.
Styling & Engine : Emotion (@emotion/react, @emotion/styled).
Architecture CSS : Pas de Tailwind. Utiliser le Theming de MUI (création d'un fichier theme.js) pour les couleurs globales et des fichiers CSS Modules ou des fichiers .css séparés pour les mises en page spécifiques et les animations complexes (Background spatial).
Icônes : @mui/icons-material.
Rends le site entièrement responsive (mobile-friendly).
2. Système de Design (Theming)
Claude, tu dois d'abord configurer un ThemeProvider avec les spécifications suivantes :
Palette : Mode dark par défaut.
Primary : Violet néon (#9c27b0 ou proche de l'image).
Secondary : Bleu cyan/électrique.
Accent : Vert émeraude (pour le bouton "Inscription").
Components Override : Personnaliser les MuiButton pour qu'ils aient des bordures dégradées et des effets de lueur (glow) comme sur la maquette.
Typographie : Utiliser une police sans-serif moderne (ex: 'Inter', 'Roboto').
3. Structure des Pages (Composants Réutilisables)
Toutes les pages doivent utiliser les mêmes composants de base issus de MUI pour assurer l'unité :
Layout Global : Un wrapper incluant la Navbar et le Footer.
Hero Section : Utiliser Grid2 (MUI) pour séparer le texte (gauche) et l'illustration mobile (droite).
Feature Cards : Utiliser le composant Card de MUI personnalisé avec un effet de verre (Glassmorphism) et des bordures néon.
Auth Pages (Login/Register) : * Créer des vues statiques avec TextField (MUI) stylisés.
Les formulaires doivent être fonctionnels visuellement (validation basique) mais sans appel API (utiliser e.preventDefault()).
4. Structure de la Page Principale (Landing) :
Navbar : Logo "RE:", liens (Accueil, L'Appli, Le Projet, Le Développeur) et boutons "S'inscrire" / "Se Connecter".
Hero Section : * À gauche : Titre "RE: La Culture, Version Privée entre Amis !" et texte descriptif. Boutons d'action "Découvrir l'appli" et "Inscription".
À droite : Mockup d'iPhone affichant l'interface de l'application (listes de films, quiz du jour, activités récentes).
Section Features (3 colonnes) : * "Re: C'est Quoi ?" (Réseau social de découvertes).
"Un Cercle Privé" (Confidentialité et amis proches).
"Qui Suis-Je ?" (Présentation du développeur).
Section Administration : Un bloc "Administrez Votre Compte" avec un visuel d'ordinateur portable et les boutons de connexion.
Footer : Mentions légales, Politique de confidentialité, Copyright 2024 et icônes sociales (X/Twitter, Instagram).
5. Pages Supplémentaires (Maquettes statiques) :
Page Inscription : Créer une page de formulaire stylisée avec les champs Nom, Email, Mot de passe. Le bouton "S'inscrire" doit être présent mais ne doit pas envoyer de données (prévoir un e.preventDefault()).
Page Connexion : Formulaire similaire pour l'email et le mot de passe.

6. Spécifications Visuelles "Cyberpunk"
Background : Implémenter un arrière-plan fixe dans le fichier CSS global avec un dégradé radial sombre et un effet de "Space Nebula" (utiliser des box-shadow diffus et des dégradés linéaires).
Animations : Intégrer des transitions douces sur les boutons et un léger effet d'entrée sur les cartes au scroll.
Police : Sans-serif moderne et épurée (type Inter ou Montserrat).
7. Organisation des Fichiers
/src/theme/ : Configuration du thème MUI.
/src/components/ : Boutons personnalisés, Navbar, Footer, Cards.
/src/pages/ : Home.jsx, Login.jsx, Register.jsx.
/src/styles/ : Fichiers .css séparés pour les styles globaux et les animations de fond.
